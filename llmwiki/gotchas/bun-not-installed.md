## Symptom

`bun install` / `bun run <script>` → `command not found: bun`

## Root cause

The GitHub Codespace used for this project ships with Node 24 + npm 11 but does not have bun installed. The project has a `bun.lock` (generated by Lovable) but `package-lock.json` is also present and kept in sync.

## Fix

All Makefile frontend targets use `npm` and `npm run`:

```makefile
install-frontend:
    cd frontend && npm install

dev-frontend:
    cd frontend && npm run dev
```

`npm run <script>` resolves to the same Vite/vitest/eslint/prettier binaries that `bun run` would use — the scripts in `package.json` are identical.

## Note

If bun is installed later (`npm install -g bun` or via the bun installer script), the Makefile can be updated back to `bun` for faster installs. The `bun.lock` file is still valid.
